fusionner is French for merge. This tool exists to create merge commits between your watched topic branches and your target default branch.

These merged commits can then be tested in your CI tool.

Normally, tests will be run on the commit labelled Pull Request. Ideally, we would like to run tests on a merge commit with your master branch. This is what fusionner does!

If your master branch has moved on, fusionner will update the merge commit with the new commits from master.

Prior Art

fusionner is inspired by bors and refs/pul/xxx/merge references that Github provides.

Usage

If you are looking for usage, refer to the repository. This documentation is intended for using fusionner as a library in your Rust application.
